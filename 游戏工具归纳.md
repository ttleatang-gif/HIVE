# 昆虫棋游戏实现工具归纳

## 一、C++ 标准库组件

### 1. 容器类（STL Containers）
- **`std::vector`** - 核心数据结构
  - 存储棋盘状态 `map`（三维向量）
  - 存储坐标位置 `{x, y, z}`
  - 存储后继位置列表
  - 自定义 `Queue` 类的底层实现

- **`std::unordered_map`** - 高效位置查找
  - 键：位置哈希值（`int`）
  - 值：该位置上的棋子列表（`vector<pair<int, int>>`）
  - 用途：快速查找某个位置上的所有棋子

- **`std::pair`** - 组合数据类型
  - 存储 `{玩家编号, 棋子编号}` 对
  - 用于 `find` 查找表中

### 2. 算法库（Algorithm）
- **`std::find`** - 查找元素
  - 在 `dfs` 函数中检查路径中是否已包含某个位置
  - 在后继列表中查找重复位置

- **`std::remove_if`** - 条件删除
  - 在 `getLegalOneSteps` 中移除特定棋子记录

### 3. 字符串处理
- **`std::string`** - 字符串操作
  - 用于哈希函数的坐标转字符串
  - `std::to_string` 进行数值转字符串
  - `std::stringstream` 用于格式化输出

### 4. 数学库
- **`<cmath>`** - 数学函数
  - `sqrt()` 计算欧几里得距离（鼠标点击位置判断）

### 5. 异常处理
- **`<stdexcept>`** - 异常类
  - `std::runtime_error` 用于队列空异常

### 6. 功能对象
- **`std::hash<std::string>`** - 哈希函数对象
  - 将坐标字符串转换为哈希值
  - 用于位置查找表的键

## 二、图形渲染库

### SFML (Simple and Fast Multimedia Library)
- **`SFML/Graphics.hpp`** - 图形渲染
  - `RenderWindow` - 游戏窗口
  - `CircleShape` - 绘制棋子（圆形）
  - `Vertex` / `Lines` - 绘制网格线
  - `Color` - 颜色定义
  - `Text` / `Font` - 文本显示
  - `Vector2f` / `Vector2i` - 二维向量

- **`SFML/Window.hpp`** - 窗口和事件管理
  - `Event` - 事件处理
  - `Mouse` - 鼠标输入
  - `VideoMode` - 视频模式设置
  - `Clock` - 时间控制（AI延迟）

## 三、核心算法

### 1. 图搜索算法
- **BFS（广度优先搜索）**
  - 实现位置：`getAntSuccessor()` (346-377行)
  - 用途：计算工蚁（Ant）的所有可能移动位置
  - 特点：确保找到所有可达位置，用于无限移动的棋子

- **DFS（深度优先搜索）**
  - 实现位置：`dfs()` (380-397行) 和 `getSpiderSuccessor()` (401-405行)
  - 用途：计算蜘蛛（Spider）的移动（必须恰好3步）
  - 特点：限制深度为3，避免重复路径

### 2. 连通性检查算法
- **实现位置**：`isConnectivity()` (491-543行)
- **算法类型**：BFS遍历
- **用途**：检查棋盘上所有棋子是否连通（Hive规则要求）
- **关键点**：
  - 使用 BFS 从起始点遍历所有相邻棋子
  - 通过查重数组标记已访问的棋子
  - 最终检查是否所有棋子都被访问

### 3. 哈希技术
- **位置哈希函数**：`hashPos()` (54-57行)
  - 输入：三维坐标 `{x, y, z}`
  - 处理：转换为字符串后哈希
  - 输出：整数哈希值
  - 用途：作为 `find` 查找表的键

## 四、自定义数据结构

### 1. Queue 类
- **实现位置**：26-49行
- **功能**：自定义队列，存储 `vector<int>` 类型元素
- **用途**：
  - BFS 算法的辅助数据结构
  - 连通性检查中的遍历队列
- **实现方式**：基于 `vector` 实现（使用 `erase` 模拟出队）

### 2. 六边形坐标系统
- **坐标偏移量**：`NEAR` (113行)
  - 6个方向的偏移量：`{0,1}, {1,1}, {1,0}, {0,-1}, {-1,-1}, {-1,0}`
  - 用于计算相邻位置

### 3. 三维坐标系统
- **坐标格式**：`{x, y, z}`
  - `x, y`：二维棋盘坐标（六边形网格）
  - `z`：高度（支持甲虫堆叠功能）

## 九、总结

### 核心依赖
1. **STL容器**：`vector`, `unordered_map`, `pair` - 数据存储和查找
2. **SFML库**：图形渲染和用户交互
3. **图算法**：BFS、DFS - 棋子移动计算
4. **哈希技术**：快速位置查找
5. **坐标系统**：六边形网格 + 三维坐标

### 关键设计模式
- **状态模式**：游戏状态管理
- **策略模式**：不同棋子的移动规则
- **查找表模式**：使用哈希表加速位置查找

### 性能优化
- 使用 `unordered_map` 实现 O(1) 位置查找
- 哈希表缓存位置信息，避免重复计算
- BFS/DFS 算法优化棋子移动计算
